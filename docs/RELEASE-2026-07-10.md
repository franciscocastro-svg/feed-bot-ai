# Release 2026-07-10

Esta release endurece autenticação, credenciais do Instagram, captação de fontes,
exclusão de dados e o fluxo de Cortes IA.

## Lovable Cloud

1. Aplicar, nesta ordem:
   - `supabase/migrations/20260710120000_harden_platform_security_and_cuts.sql`;
   - `supabase/migrations/20260710170000_auto_approve_verified_users.sql`;
   - `supabase/migrations/20260710183000_isolate_account_template_defaults.sql`.
2. Redeploy das funções:
   - `auth-email-hook`
   - `discover-rss`
   - `fetch-rss`
   - `preview-source`
   - `fetch-insights`
   - `instagram-data-deletion`
   - `keep-ig-token-alive`
   - `meta-usage-refresh`
   - `process-news`
   - `publish-scheduler`
   - `refresh-ig-token`
   - `verify-ig-token`
3. Se o Send Email Hook estiver configurado como webhook bruto do Supabase, criar
   `AUTH_EMAIL_HOOK_SECRET` e enviar o mesmo valor no header
   `x-auth-email-hook-secret`. O webhook assinado da Lovable continua compatível.
4. O cadastro passa a usar código de seis dígitos. E-mail confirmado aprova o
   cadastro automaticamente, mas o painel continua exigindo assinatura ativa ou
   período de teste iniciado por pagamento.
5. Templates passam a ser resolvidos exclusivamente pela conta Instagram
   selecionada. O padrão global não é herdado entre contas de um cliente com
   múltiplos Instagrams.

## VPS

O deploy agora troca o processo antigo `feedbot-worker` por:

- `feedbot-cuts`: downloads, transcrição e renderização de cortes;
- `feedbot-media`: artes, Reels e mídia das publicações;
- `feedbot-webhook`: deploy automático já existente.

Na pasta `/opt/feedbot`, executar `bash scripts/deploy-vps.sh`. O script instala,
testa, compila, valida o worker e reinicia os processos pelo
`ecosystem.config.cjs`.
